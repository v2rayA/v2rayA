# v2rayA 产品上下文

## 解决的问题
1. **V2Ray 配置复杂性**: 原生 V2Ray 需要手动编写 JSON 配置文件，对普通用户不友好
2. **跨平台支持**: 需要在不同操作系统上实现统一的代理管理体验
3. **透明代理配置**: Linux 下配置透明代理需要复杂的 iptables 规则
4. **订阅管理**: 需要方便地管理多个代理服务器和订阅源

## 工作原理
1. **Web GUI 架构**: 前端提供 Web 界面，后端 Go 服务管理 v2ray-core
2. **代理核心管理**: 动态生成 v2ray-core 配置并管理其生命周期
3. **透明代理**: 通过 iptables/nftables 规则实现流量重定向
4. **系统代理**: 在 Windows/macOS 上配置系统级代理设置

## 用户体验目标
- **简单易用**: 通过 Web 界面一键配置代理
- **远程管理**: 可部署在路由器或 NAS 上，远程访问管理
- **自动化**: 支持订阅自动更新、GFWList 自动更新
- **灵活性**: 支持多种协议、自定义路由规则

## 目标用户
1. 需要科学上网的普通用户
2. 需要部署家庭网络代理的技术用户
3. 需要在服务器/路由器上运行代理的系统管理员
